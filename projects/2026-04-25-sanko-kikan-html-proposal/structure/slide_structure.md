# Slide Structure

## 01. Cover

- Role:
  提案テーマと資料の温度感を最初に伝える。
- Core message:
  今回の提案は、日常業務の効率化と将来的な展望まで見据えた基幹管理システム提案である。
- Primary visual:
  既存参考資料の表紙デザインを踏襲した、余白のある強い表紙。
- Required copy:
  - 日常業務の効率化と将来的な展望を見据えた
  - 基幹管理システムのご提案

## 02. Agenda

- Role:
  この資料で何を順番に確認するかを、自然な対外向け表現で示す。
- Core message:
  本資料は、提案の骨子、デモ画面、将来展開と進め方を確認するための構成である。
- Primary visual:
  既存参考資料の目次デザイン踏襲。
- Required copy:
  - 今回のご提案につきまして
  - 画面イメージ
  - 将来的な展望と今後の進め方

## 03. Divider: 今回のご提案につきまして

- Role:
  提案本編の入口として、今回のご提案で何をお見せするかを自然に切り替える。
- Core message:
  現在の切り替え検討を前提に、今回ご相談いただいている要件とご提案内容を順に確認いただく章である。
- Primary visual:
  フルレッドの差し込み。
- Required copy:
  - 今回のご提案につきまして

## 04. 現状課題

- Role:
  現場ヒアリングベースで、今回見えている課題を事実ベースで整理する。
- Core message:
  現場に定着しやすい設計、日常業務の手間、スマホ現場対応、他社物件出稿が今回の主要課題である。
- Primary visual:
  4カード + 下部の結論メッセージ。
- Required copy:
  - 現場に定着しやすい設計
  - 日常業務の手間（修繕・契約書作成等）
  - スマホ現場対応
  - 他社物件の出稿

## 05. 今回のご提案内容

- Role:
  どんな機能を並べるかではなく、どのような業務状態を目指す提案かを示す。
- Core message:
  基幹管理システムを中核に、日常業務を一つの流れとして扱える状態をつくることが今回の提案の骨子である。
- Primary visual:
  強いリード文 + 4カード構成。
- Required copy:
  - 基本機能を一つの基盤に整理
  - 契約書類と請求対応を標準化
  - 現場利用しやすいUIとスマホ対応
  - AI活用までつながる拡張性

## 06. Divider: デモ画面イメージ

- Role:
  抽象的な提案内容から、具体的な運用イメージへ移る。
- Core message:
  実際の画面イメージを通じて、要件と運用の具体像を確認する。
- Primary visual:
  フルレッドの差し込み。
- Required copy:
  - デモ画面イメージ

## 07. デモ画面イメージ 1

- Role:
  基本機能の中でも、日常的に確認頻度の高い情報系の画面を見せる。
- Core message:
  物件、入居者、空室状況、履歴などをまとめて確認できることが、日常運用の起点になる。
- Primary visual:
  メインスクリーンショット + 注釈。
- Required copy:
  - 物件情報の確認
  - 入居者情報の確認
  - 空室状況、修繕状況、対応履歴の確認
  - 日常業務の確認起点となる一覧、詳細画面

## 08. デモ画面イメージ 2

- Role:
  契約、請求、インボイス対応と、現場でのスマホ利用をまとめて見せる。
- Core message:
  契約書類や請求実務への対応に加えて、現場でスマホから必要情報を確認、対応できることが、実運用上の重要要件である。
- Primary visual:
  複数スクリーンショット + 機能ラベル。
- Required copy:
  - 契約書等作成機能
  - 一時金請求書、家賃明細、各種インボイス対応
  - 一般契約、定期借家、法人契約など複数パターンへの対応
  - 請求先分岐や記載整理など実務上の複雑さへの対応
  - スマホ対応
  - 内見中の確認、メール返信、社内向け情報参照

## 09. デモ画面イメージ 3

- Role:
  出稿、反響対応、AI補助など周辺実務まで含めて見せる。
- Core message:
  出稿業務は必須要件であり、反響対応やAI補助まで含めて、一続きの運用として扱えることが重要である。
- Primary visual:
  4分割スクリーンショット、または複数スクリーンショット + ラベル。
- Required copy:
  - 自社物件の出稿機能
  - 他社物件の出稿機能
  - 反響取り込み、メール返信効率化
  - AIによる返信文作成、日程調整
  - AIによる各種データ検索

## 10. Divider: 将来的な展望と今後の進め方

- Role:
  画面紹介の先にある将来像と、現実的な進め方に話をつなぐ。
- Core message:
  今回の提案は、目先の業務効率化だけでなく、将来的な展望まで見据えたものである。
- Primary visual:
  フルレッドの差し込み。
- Required copy:
  - 将来的な展望と今後の進め方

## 11. 展開イメージ

- Role:
  基幹データを整える意味を説明する。
- Core message:
  基幹データを整えることで、AI返信、検索、営業支援などの展開が現実的になる。
- Primary visual:
  中央ハブ + 周辺カード。
- Required copy:
  - 基幹データ
  - 反響対応のAI支援
    - 反響内容や物件情報をもとに、返信文作成や日程調整を補助できる
  - 社内データ横断検索
    - 物件、顧客、対応履歴、契約関連情報を横断して確認できる
  - 将来的な営業支援や書類活用への展開
    - 訪問時の提案支援や、契約、社内文書作成の補助へ広げられる

## 12. 今後の進め方

- Role:
  導入までの流れを、参考資料の進め方そのままの内容で見せる。
- Core message:
  NDA各種契約締結から、進め方確認、移行先準備、開発、現場導入、改善までを段階的に進める。
- Primary visual:
  横並びステップ型ロードマップ。
- Required copy:
  - ご契約
  - 進め方の確認
  - データ移行先の準備
  - エージェント開発
  - 現場導入
  - 改善

## 13. 補助金活用について

- Role:
  AIを含めた開発に進む場合の補助金活用可能性と支援内容を補足する。
- Core message:
  AIを含めた開発に進む場合は、補助金活用の可能性と申請支援もあわせて確認できる。
- Primary visual:
  補助金活用の可能性、条件、支援内容を整理した補足スライド。
- Required copy:
  - 中小企業省力化投資補助金（一般型）の活用可能性
  - 条件次第で最大6,500万円
  - 申請可否の確認や資料整理のサポート
