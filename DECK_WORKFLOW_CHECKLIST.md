# 資料作成チェックリスト

## 0. 開始前

- [ ] `direction.md` または要件定義書がある。
- [ ] 方向性から始めた場合、`requirements.md` と `intake/assumptions.md` がある。
- [ ] 参考資料がある。
- [ ] ロゴ・必須画像がある。
- [ ] 出力形式が決まっている。

## 1. 戦略

- [ ] ワンメッセージが1文で言える。
- [ ] 読み手の判断が明確。
- [ ] サブメッセージが3-6個に整理されている。

## 2. 構成

- [ ] 章構成が決まっている。
- [ ] 各スライドの役割が重複していない。
- [ ] 不要な課題スライドなどを削っている。

## 3. コピー

- [ ] 見出しが短い。
- [ ] 本文が意味単位で改行されている。
- [ ] 用語が統一されている。

## 4. デザイン

- [ ] サイドバー仕様が統一されている。
- [ ] 章表示が本文エリアにある。
- [ ] アイコンが背景円の中心に見える。
- [ ] スクショ枠が十分大きい。

## 5. 実装

- [ ] `index.html` は1スライド1section。
- [ ] `styles.css` に共通ルールがまとまっている。
- [ ] 画像は `assets/` 配下。
- [ ] スクショ差し込み場所がある。

## 6. 最終確認

- [ ] 右端・下端の見切れがない。
- [ ] 文字が読める。
- [ ] 赤帯や装飾がズレていない。
- [ ] PDF/PNG出力で崩れない。
- [ ] 修正した微調整ルールを `DECK_MICRO_ADJUSTMENT_RULES.md` に追記した。
