# Deck Structure Planner

## Goal

Decide what each slide must do and how the reader moves through the deck.

## Process

1. Start with one-message and submessages.
2. Choose the minimum slide count that can prove the message.
3. For each slide, define:
   - slide title
   - role in the argument
   - primary visual type
   - required copy
   - reference section to consult for design
4. Remove slides that only say "we will provide" or repeat the same point.
5. Store output in `structure/slide_structure.md`.

## Preferred Slide Roles

- cover
- agenda
- chapter divider
- current context
- proposal summary
- mock UI
- roadmap
- comparison
- future expansion
