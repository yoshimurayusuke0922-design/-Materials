# Deck One Message

## Goal

Reduce the deck to one sentence the reader should remember.

## Process

1. Identify the decision context: who reads it, what they need to decide, and what anxiety they have.
2. Separate the offer from the thesis. A product name is not the message.
3. Write one sentence in this shape:
   - `X is not just A; it is B that enables C.`
   - or `Use X as the starting point to achieve Y and prepare for Z.`
4. Reject messages that are:
   - a feature list
   - a vague benefit
   - a vendor-centered claim
   - too broad to prove in the deck
5. Store the final message in the project `strategy/one_message.md`.

## Output

- One main message.
- Why this is the right message.
- What the deck should not try to say.
