# Deck Submessage Map

## Goal

Create the small set of claims that prove the one-message.

## Process

1. Start from the one-message, not from slide count.
2. Write 3-6 submessages. Each must prove one part of the main message.
3. Keep each submessage as a claim, not a topic label.
4. For each submessage, define:
   - supporting evidence or artifact
   - slide type that can prove it
   - what not to overclaim
5. Store the result in `strategy/submessages.md`.

## Quality Bar

- The set must be mutually distinct.
- No submessage should depend on a later unexplained concept.
- Avoid internal convenience words such as "稟議で通しやすい"; keep the reader-facing value neutral and concrete.
