# choose-static-implementation

## 目的

今回の資料に最適な静的コード実装方式を選定し、理由を記録する。

## 入力

- 入力画像一覧
- スライド枚数
- 編集しやすさの要件
- 出力先とプレビュー要件

## 出力

- `work/implementation-choice.md`

## 手順

1. 素のHTML/CSS/JavaScript、Astro、React、Svelte、MDX、Reveal.js、Slidev、SVG中心実装を比較する。
2. 導入負荷、編集性、壊れにくさ、スクリーンショット化のしやすさで評価する。
3. 選んだ方式、選ばなかった方式、編集対象ファイルを記録する。
